# Scout Ultra: Отчет по проверке качества (5W+H Challenge)

## Методология проверки
Протокол: **5W+H Challenge** (What, Why, Who, When, Where, How)  
Целевая оценка: **9.8/10**  
Дата проверки: 2026-01-22

---

## 1. WHAT (Что создано?)

### ✅ Что ЕСТЬ:
- 8 логичных эпиков, покрывающих весь функционал Scout Ultra
- Детальная разбивка каждого эпика на компоненты
- Definition of Done для каждого эпика
- Календарный план на 27 недель с фазами
- Учет груминга и дизайна в роадмапе
- Метрики успеха по фазам
- Риски и митигация

### ❌ Что ОТСУТСТВУЕТ:
1. **Конкретные дизайн-задачи из PRD Раздел 8:**
   - WF-C01: Точки входа (Zero-State, Pricing, Booking Modal)
   - WF-C02: Progress Widget (Timer, Stepper, Ticker)
   - WF-C03: Enriched Shortlist (бейджи, видео, Scout's Note)
   - WF-C04: Negative Scenarios UI (SLA Breach, Revision Modal, Rating)
   - WF-A01: Admin Queue Dashboard
   - WF-A02: SR Workbench (3-колоночная верстка)
   - WF-A03: Enrichment Modal
   
   **В роадмапе:** только общие упоминания "Hi-Fi макеты", "UI Review"

2. **Детализация Negative Scenarios (PRD Раздел 9):**
   - 9.1. SLA Breach сценарий (компенсация, эскалация)
   - 9.2. Rejection Loop (клиент отклонил всех кандидатов)
   - 9.3. Rating & Quality Control
   - 9.4. Data Security & GDPR compliance
   
   **В роадмапе:** частично в EPIC 7, но не как отдельные задачи

3. **Операционный план:**
   - Найм и обучение первых Orchestrators
   - SOP (Standard Operating Procedures) для менеджеров
   - Скрипты коммуникации с кандидатами
   - Playbook для разных типов запросов

4. **План коммуникации с пользователями:**
   - Анонс функционала
   - Onboarding для Explicit Mode
   - Email-кампании для активации
   - Help Center статьи

5. **Техническая документация:**
   - API спецификации для интеграций
   - Data Migration план (если нужна)
   - Rollback стратегия

**Оценка WHAT: 8.0/10** (основное есть, но критические детали упущены)

## 2. WHY (Зачем? Бизнес-ценность)

### ✅ Что ЕСТЬ:
- Каждый эпик имеет "Ценность для бизнеса"
- Метрики успеха определены по фазам
- Общая стратегия запуска объяснена (Value First, Risk Mitigation)

### ❌ Что ОТСУТСТВУЕТ:
1. **Связь с метриками из PRD Раздел 1:**
   - Zero-Result Rate Reduction (цель: <20% за 24ч, <1% за 48ч)
   - SLA Compliance (100% для Ultra)
   - Outreach Efficiency (конверсия Invite → Response)
   - Unit Economics Attribution
   
   **Проблема:** В роадмапе метрики есть, но они не привязаны к конкретным эпикам

2. **Ожидаемые бизнес-результаты:**
   - Сколько SR должен закрыть 1 Orchestrator в неделю?
   - Какая целевая конверсия Implicit → Explicit?
   - Прогноз выручки от Ultra в первые 3 месяца

3. **ROI обоснование:**
   - Стоимость разработки vs прогнозируемая выручка
   - Break-even point

**Оценка WHY: 8.5/10** (ценность ясна, но нет финансовых прогнозов)

## 4. WHEN (Когда? Временные рамки)

### ✅ Что ЕСТЬ:
- Детальный календарный план по неделям (0-27)
- Груминг расписан за 2-4 недели до разработки
- Дизайн на 1-2 недели опережает dev
- Фазы с четкими milestone

### ❌ Потенциальные проблемы:

1. **Недостаточные буферы:**
   - EPIC 2 (Admin Console): оценка 4-5 недель, выделено 4 недели (риск срыва)
   - Нет запаса на баги, переработки, код-ревью
   - Recommended: +20% буфер на каждую фазу

2. **Праздники и отпуска:**
   - 27 недель = ~6.5 месяцев
   - Нет учета майских праздников, летних отпусков
   - Recommended: календарная сверка с праздниками

3. **Зависимости внешних команд:**
   - Интеграция с Calendly/Loom может затянуться (API limits, approvals)
   - Stripe integration требует compliance review (может добавить 1-2 недели)

4. **Отсутствие contingency плана:**
   - Что если ключевой разработчик уволится?
   - Что если дизайн не утвердят с первого раза?

**Рекомендация:** Добавить Phase 5.5 (2 недели) как буфер перед Full Launch

**Оценка WHEN: 8.5/10** (хороший план, но нет буферов)

## Критические пропуски (Must Fix)

### 1. Дизайн-задачи не детализированы (Раздел 8 PRD)
**Проблема:** В PRD есть 7 конкретных wireframes (WF-C01 - WF-C04, WF-A01 - WF-A03) с детальным описанием элементов. В роадмапе только общие упоминания "дизайн макетов".

**Решение:** Создать отдельный EPIC 9: Design System & Wireframes OR детализировать в каждом эпике конкретные макеты.

### 2. Negative Scenarios не превращены в задачи (Раздел 9 PRD)
**Проблема:** SLA Breach, Rejection Loop, Rating System, Security/GDPR — это отдельные фичи, требующие dev/design/QA.

**Решение:** Расширить EPIC 7 или создать подзадачи в существующих эпиках.

### 3. Операционный план отсутствует
**Проблема:** Нет плана найма, обучения, SOP для Orchestrators. Без этого функционал не заработает.

**Решение:** Добавить Phase 0.5: Operations Prep (найм, обучение, SOP).

### 4. Коммуникационный план отсутствует
**Проблема:** Как пользователи узнают о Ultra? Как онбордим клиентов на платный режим?

**Решение:** Добавить раздел "Go-to-Market Plan" в роадмап.
